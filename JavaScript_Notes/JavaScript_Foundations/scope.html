Understanding Scope in JavaScript: Where Variables Live & Die

Step 1: What is Scope?

Scope determines where variables are accessible in your code. Think of it as the “visibility rules” of variables.

Mental model:

“Variables exist inside boxes. Some boxes are nested inside others. You can see variables in your own box or outer boxes, but not in inner boxes unless passed.”

Step 2: Global Scope

Variables declared outside any function or block are in the global scope. They can be accessed anywhere in your code. 

let globalVar = "I am global";

function showGlobal() {
  console.log(globalVar); // accessible
}

showGlobal();
console.log(globalVar); // also accessible
Mini Challenge: Declare a global variable and access it from two different functions.

Step 3: Function Scope

Variables declared inside a function are in function scope. They can only be accessed inside that function. 

function myFunction() {
  let funcVar = "I am local to this function";
  console.log(funcVar); // works
}

myFunction();
console.log(funcVar); // ❌ Error: funcVar is not defined

Mental model: Function scope = a sealed box. Variables live inside the box and disappear when the function ends.

Mini Challenge: Write a function with a local variable, then try to access it outside the function. Observe the error.

Step 4: Block Scope

Variables declared with let or const inside { } blocks are in block scope. Variables declared with var are function-scoped, not block-scoped. 

if (true) {
  let blockVar = "I exist only in this block";
  var functionVar = "I exist in the whole function";
  console.log(blockVar); // ✅ works
}

console.log(blockVar); // ❌ Error
console.log(functionVar); // ✅ works if inside the same function
Mini Challenge: Compare let, const, and var inside a block. Which are accessible outside?

Step 5: Lexical / Static Scope

JavaScript uses lexical scope: a function can access variables from its outer scopes where it was defined, not where it is called. 

let outer = "I am outer";

function parent() {
  let inner = "I am inner";

  function child() {
    console.log(outer); // ✅ can access outer
    console.log(inner); // ✅ can access inner
  }

  child();
}

parent();

Mental model: Variables are visible in nested boxes up the chain, but not in sibling or parent boxes.

Mini Challenge: Create a nested function that logs a variable from the outer function. Try logging an inner variable from the outer function — what happens?
