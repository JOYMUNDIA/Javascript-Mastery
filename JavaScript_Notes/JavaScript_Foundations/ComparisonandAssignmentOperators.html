From Zero Knowledge → Confident JavaScript Developer

Step 1: What Are Operators?

Think of operators as instructions for your variables. They tell JavaScript to perform a specific action on one or more values.

Just like math in real life, JavaScript has operators to:

  • Compare values
  • Do calculations
  • Combine logic
  • Store or update values
let a = 5;
let b = 10;
let sum = a + b; // + is the addition operator

Here, + tells JavaScript to add the numbers stored in a and b.

Mini Challenge: Try using -, *, and / with two numbers and see the results in the console.

Step 2: Comparison Operators

Comparison operators ask questions about relationships between two values.

Every comparison always returns a boolean value: 

true
false

Think of comparison operators like a security guard asking questions before allowing entry.

Example questions a program might ask:

  • Is the user old enough?
  • Is the password correct?
  • Does the account have enough money?
  • Is the score high enough to pass?

These questions are answered using comparison operators.

---

Loose Equality (==)

The == operator checks if two values are equal, but JavaScript may convert data types automatically. 

5 == "5"

JavaScript converts the string into a number and compares them. 

true

Analogy:

Imagine a teacher grading homework.

  • Student A writes 5
  • Student B writes "5"

The teacher says: "Both answers represent five", so they are considered equal.

---

Strict Equality (===)

Strict equality checks both value AND type. 

5 === "5"

Now JavaScript asks two questions:

  • Are the values the same?
  • Are the types the same?

Because one is a number and the other is a string, the result is: 

false

Analogy:

Airport security checks both:

  • The person's identity
  • The authenticity of their passport

If anything doesn't match, entry is denied.

---

Not Equal (!=)

The != operator checks if two values are different. 

5 != 3

Result: 

true

Because 5 and 3 are not the same.

Analogy:

A teacher checks if a student's name is on the absent list. 

studentName != absentList
---

Strict Not Equal (!==)

Strict not equal checks both value and type. 

5 !== "5"

Result: 

true

Because one is a number and the other is a string.

---

Greater Than (>)

Checks if the value on the left is larger than the value on the right. 

10 > 5

Result: 

true

Analogy:

A bank checks if you have enough money before a withdrawal. 

balance > withdrawalAmount
---

Less Than (<)

Checks if the value on the left is smaller. 

3 < 7

Result: 

true

Analogy:

A police officer checking if a driver is under the speed limit. 

speed < limit
---

Greater Than or Equal (>=)

This checks if a value is either greater than OR equal to another value. 

5 >= 5

Result: 

true

Analogy:

A job posting may require: 

experience >= 3 years
---

Less Than or Equal (<=)

Checks if a value is smaller than OR equal to another value. 

age <= 12

Used for child tickets or age limits.

---

Real Program Example

Comparison operators are most powerful when used with decision making. 

let age = 16;

if (age >= 18) {

} else {
console.log("You are too young");
}

Step 3: Logical Operators

Logical operators combine multiple conditions to make smarter decisions in programs.

Think of logical operators like combining multiple questions together.

Example real-life questions:

  • Is the person an adult AND do they have an ID?
  • Is the password correct OR does the user have recovery access?
  • Is the user NOT banned?

Logical operators combine these conditions and return a boolean value: 

true
false
---

The Three Logical Operators

Operator Name Meaning
&& AND Both conditions must be true
|| OR At least one condition must be true
! NOT Reverses a boolean value
---

Logical AND (&&)

The AND operator returns true only if BOTH conditions are true. 

true && true
Result: 
true
But if one condition fails: 
true && false
Result: 
false
---

Analogy: Club Entrance

A security guard allows entry only if:

  • You are 18 or older
  • You have valid ID
Both conditions must be satisfied.

---

Logical OR (||)

The OR operator returns true if at least one condition is true. 

true || false
Result: 
true
---

Analogy: Login System

A website may allow login if you provide:

  • Password
  • OR fingerprint
Only one is required. 

passwordCorrect || fingerprintMatch
Example:

let fingerprintMatch = true;

console.log(passwordCorrect || fingerprintMatch);
Result: 
true
Because one condition is true. ---

Logical NOT (!)

The NOT operator reverses a boolean value.

Original NOT Result
true false
false true

!true
Result: 
false

Analogy: Banned Users

A website allows login only if the user is NOT banned. 

!isBanned
Example:

console.log(!isBanned);
Result: 
true
User is allowed because they are not banned. ---

Combining AND + OR

Programs often mix logical operators. Example: Special event entry. Rules:
  • Must be 18+

  • OR must be staff

console.log("Entry allowed");
}
Because the person is staff, entry is allowed. ---

Step 5: Assignment Operators — Updating the Cube

Earlier we learned that a variable is like a cube in memory.

The cube has:

  • a label (the variable name)
  • a value inside it

Example: 

let score = 10;

JavaScript creates a cube labeled score and puts the value 10 inside.


score
10

Assignment operators are simply shortcuts for updating what is inside the cube.

The Basic Assignment Operator (=)

The = operator replaces whatever is inside the cube. 

score = 20;

JavaScript does this:

  • 1️⃣ Find the cube labeled score
  • 2️⃣ Remove the value 10
  • 3️⃣ Place 20 inside

score
20

Addition Assignment (+=)

The += operator means:

“Take the value already inside the cube, add something to it, and store the result back in the same cube.”
Example: 
let score = 10;

score += 5;

JavaScript interprets this as: 

score = score + 5;

Step by step:

  • 1️⃣ Read the value inside the cube → 10
  • 2️⃣ Add 5
  • 3️⃣ Store the result 15 back in the cube

score
15

Subtraction Assignment (-=)

The -= operator subtracts a value from the cube.

Example: 
let lives = 5;

lives -= 1;
Equivalent to: 
lives = lives - 1;

JavaScript:

  • 1️⃣ Reads the value inside the cube → 5
  • 2️⃣ Subtracts 1
  • 3️⃣ Stores the result 4 back in the cube

lives
4

Multiplication Assignment (*=)

This multiplies the cube's value and stores the result.

Example: 
let money = 50;

money *= 2;
Equivalent to: 
money = money * 2;

Cube update:

  • 50 × 2 = 100
  • The cube now contains 100

money
100

Division Assignment (/=)

Example: 
let pizza = 12;

pizza /= 3;
Equivalent to: 
pizza = pizza / 3;

The cube updates from 12 to 4.


pizza
4

Remainder Assignment (%=)

This stores the remainder after division. Example: 
let candies = 10;

candies %= 3;
Equivalent to: 
candies = candies % 3;

Result stored in cube: 

1

candies
1

Exponent Assignment (**=)

Raises the cube value to a power. Example: 
let number = 2;

number **= 3;
Equivalent to: 
number = number ** 3;

Result stored in the cube: 

8

number
8

Complete Assignment Operator Table

Operator Meaning Equivalent Code
= Assign value x = 5
+= Add and store x = x + 5
-= Subtract and store x = x - 5
*= Multiply and store x = x * 5
/= Divide and store x = x / 5
%= Remainder and store x = x % 5
**= Power and store x = x ** 5

Step 6: Combining Operators

In real programs, we rarely use just one type of operator. Most decisions require combining comparison operators, logical operators, and variables.

This is how programs make intelligent decisions.

Example Program

let age = 20;
let hasLicense = true;

let canDrive = (age >= 18) && hasLicense;

console.log(canDrive);

This program checks whether someone is allowed to drive.

Step 1: Create Variables (Cubes in Memory)

let age = 20;
let hasLicense = true;

JavaScript creates two cubes in memory.

  • A cube labeled age containing 20
  • A cube labeled hasLicense containing true

These cubes store information about a person.

Step 2: Comparison Operator

Inside our expression we see: 
age >= 18

This uses the greater than or equal operator.

JavaScript asks the question:

Is the value inside the age cube greater than or equal to 18?

The value is: 

20 >= 18
Result: 
true

So the first condition becomes: 

true

Step 3: Logical Operator

Now look at the full expression: 
(age >= 18) && hasLicense
After the comparison is evaluated, JavaScript effectively sees: 
true && true

The logical AND operator && checks if both conditions are true.

Truth table reminder:
A B A && B
true true true
true false false
false true false
false false false
Since both values are true: 
true && true
Result: 
true

Step 4: Assignment Operator

Now the result is stored in a new variable: 
let canDrive = true;
JavaScript creates another cube labeled: canDrive And places the value: true inside it.

Step 5: Output the Result

Finally we run: 
console.log(canDrive);
The console prints: 
true
Meaning:
The person is allowed to drive.

How JavaScript Evaluates the Expression

JavaScript evaluates the expression step by step. Original expression: 
(age >= 18) && hasLicense
Step 1 — comparison: 
true && hasLicense
Step 2 — replace variable: 
true && true
Step 3 — logical operation: 
true

Real World Analogy

Imagine a driving school checking if someone can drive. The instructor asks two questions:
  • Is the person at least 18?
  • Do they have a license?
If the answer to both questions is yes: 
true && true
The instructor says:
"You are allowed to drive."

Changing the Values

Let's change the inputs. Example: 
let age = 16;
let hasLicense = true;

let canDrive = (age >= 18) && hasLicense;
Now JavaScript evaluates: 
16 >= 18
Result: 
false
So the expression becomes: 
false && true
Result: 
false
Meaning:
The person cannot drive because they are too young.

Another Scenario

let age = 22;
let hasLicense = false;
Evaluation: 
22 >= 18
Result: 
true
Expression becomes: 
true && false
Result: 
false
Meaning:
The person is old enough but does not have a license.
