From Zero Knowledge → Confident JavaScript Developer

Step 1: What Is Control Flow?

Control flow is the order in which JavaScript executes code. By default, JS reads code line by line from top to bottom.

Sometimes we want to make decisions, repeat code, or skip lines — that’s where control flow comes in. 

console.log("Start");
console.log("Middle");
console.log("End");

This outputs in order: Start → Middle → End.

But what if we want to skip "Middle" sometimes? That’s where conditions come in.

Step 2: Conditional Statements

1️⃣ if Statement

The if statement tells JavaScript: “Do something only if a condition is true.” 

let age = 20;

if (age >= 18) {
  console.log("You are an adult.");
}

- The condition age >= 18 is evaluated.
- If it’s true, the code inside the braces { } runs.
- Otherwise, JS skips it.

Mini Challenge: Check if a number is positive. If yes, print “Positive number”.

2️⃣ if…else Statement

Sometimes we want to do one thing if the condition is true, and another if it’s false. 

let age = 16;

if (age >= 18) {
  console.log("You are an adult.");
} else {
  console.log("You are a minor.");
}

Now JS chooses between two paths based on the condition.

3️⃣ else if Chain

Use else if when you have multiple possibilities. 

let score = 85;

if (score >= 90) {
  console.log("Grade A");
} else if (score >= 75) {
  console.log("Grade B");
} else if (score >= 60) {
  console.log("Grade C");
} else {
  console.log("Fail");
}

JS checks conditions in order and executes the first one that is true.

Mini Challenge: Write an if…else if…else to categorize age groups: child, teen, adult, senior.

Step 3: Switch Statement

The switch statement is a cleaner way to check multiple discrete values. 

let day = "Tuesday";

switch(day) {
  case "Monday":
    console.log("Start of the week");
    break;
  case "Tuesday":
    console.log("Second day");
    break;
  case "Friday":
    console.log("Almost weekend");
    break;
  default:
    console.log("Some other day");
}

- JS compares day with each case.
- break stops JS from checking the next cases.
- default runs if none of the cases match.

Mini Challenge: Use a switch statement to print a message for each season: "Winter", "Spring", "Summer", "Fall".

Step 4: Loops – Repeating Code

1️⃣ for Loop

The for loop repeats code a fixed number of times. 

for (let i = 1; i <= 5; i++) {
  console.log("Iteration " + i);
}

- i = 1 → start value
- i <= 5 → condition to continue
- i++ → increment after each loop

2️⃣ while Loop

Repeats code as long as a condition is true. 

let count = 1;

while (count <= 5) {
  console.log("Count is " + count);
  count++;
}

3️⃣ do…while Loop

Runs the code at least once, then checks the condition. 

let count = 1;

do {
  console.log("Count is " + count);
  count++;
} while (count <= 5);

Step 5: Loop Control – break & continue

break

Stops the loop completely:

for (let i = 1; i <= 10; i++) {
  if (i === 5) break;
  console.log(i);
}
// Prints 1 2 3 4

continue

Skips current iteration, continues with next:

for (let i = 1; i <= 5; i++) {
  if (i === 3) continue;
  console.log(i);
}
// Prints 1 2 4 5
Mini Challenge: Print numbers 1–10 but skip multiples of 3 using continue.

Step 6: Nested Control Flow

You can put conditions and loops inside each other to create complex logic. 

let num = 7;

if (num > 0) {
  for (let i = 1; i <= num; i++) {
    if (i % 2 === 0) {
      console.log(i + " is even");
    } else {
      console.log(i + " is odd");
    }
  }
}

- Outer if checks positivity.
- Inner for loops through numbers.
- Nested if checks odd/even.

Mini Challenge: Write a nested loop that prints a multiplication table for numbers 1–5.

Step 7: Mental Model – The Roadmap

Think of control flow as traffic lights and road signs for your code:

  • if / else: make decisions at intersections.
  • switch: choose one of many roads quickly.
  • loops: drive in a circle repeatedly.
  • break / continue: skip or stop certain streets.
