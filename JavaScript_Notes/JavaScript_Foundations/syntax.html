From Zero Knowledge → Confident JavaScript Developer

Step 1: Console Output with console.log()

The easiest way to see what your code is doing is to print messages to the console. 

console.log("Hello, world!");
console.log(42);
console.log(true);

- Think of console.log() as a “debug window” where your program tells you what’s happening.
- You can print strings, numbers, booleans, and even variables: 

let name = "Alice";
console.log("Hello " + name);
Mini Challenge: Print your favorite color and your age in two separate console.log() statements.

Step 2: User Input with prompt()

JavaScript can ask the user to type something using prompt(). 

let name = prompt("What is your name?");
console.log("Hello, " + name + "!");

- The value returned by prompt is always a string.
- You can store it in a variable and use it later.

Mini Challenge: Ask the user for their favorite number and print it back with console.log().

Step 3: Converting Input

Since prompt() always returns a string, you need to convert it for calculations. 

let num1 = Number(prompt("Enter a number:"));
let num2 = Number(prompt("Enter another number:"));
console.log("The sum is: " + (num1 + num2));

- Number() converts a string to a number.
- Other conversions: String(), Boolean().

Step 4: Comments

Comments are notes you leave in your code. They don’t run, but help you and others understand the code.

Single-Line Comments

// This is a single-line comment
console.log("Hello"); // Comment after code

Multi-Line Comments

/*
This is a multi-line comment.
You can write multiple lines here.
*/
Mini Challenge: Add a comment above each console.log statement explaining what it does.

Step 5: Doc Strings / Documentation Comments

Doc strings are special multi-line comments that explain functions, parameters, and purpose. 

/**
 * Adds two numbers and returns the result
 * @param {number} a - first number
 * @param {number} b - second number
 * @returns {number} sum of a and b
 */
function add(a, b) {
  return a + b;
}

console.log(add(5, 7)); // 12

- Useful for teams or when writing libraries.
- Many IDEs (VS Code, WebStorm) can read doc strings and show hints.

Mini Challenge: Write a doc string for a function that multiplies two numbers.

Step 6: Mental Model – Inputs and Outputs

Think of your program as a machine:

  • Inputs: prompt() or function parameters
  • Engine: your code logic
  • Outputs: console.log() or return

Comments are like labels on the machine — they don’t affect how it works but help you and others understand it.
