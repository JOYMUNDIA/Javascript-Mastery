Step 1: What is a Number in JavaScript?

In JavaScript, a number is not just “a number”.

It is a value stored inside memory that can be:

  • Whole (integer)
  • Decimal (floating point)
  • Very large
  • Very small
  • Even… invalid (NaN)
Integer
5
Decimal
3.14
NaN

Step 2: The "Box in Memory" Model

Every number lives inside a variable (your “cube” concept). 

let score = 10;

JavaScript does this:

  • Create a box labeled score
  • Store the number 10 inside

When math happens, JavaScript:

  • Reads values from boxes
  • Performs an operation
  • Stores a result (new or updated box)
---

Example

let a = 10;
let b = 3;

let result = a + b;

Step-by-step:

  • Read value inside a → 10
  • Read value inside b → 3
  • Add them → 13
  • Store inside result
---

Step 3: Types of Numbers

1. Integers (Whole Numbers)

let count = 5;
let temperature = -10;

No decimal points.

---

2. Floating Point Numbers (Decimals)

let price = 19.99;
let pi = 3.14;

These are NOT exact (important later).

---

⚠️ Hidden Truth (VERY IMPORTANT)

JavaScript stores ALL numbers using something called: 

64-bit floating point (IEEE 754)

Meaning:

  • Even integers are stored like decimals internally
  • Some numbers cannot be represented exactly
---

Example Problem

console.log(0.1 + 0.2);
Output: 
0.30000000000000004

This is NOT a bug.

This is how computers represent decimals.

---

Step 4: Special Number Values

NaN (Not a Number)

"hello" * 5
Result: 
NaN

Means:

❌ JavaScript tried to do math, but failed.

---

Infinity

10 / 0
Result: 
Infinity

JavaScript does NOT crash.

It returns a special value.

---

Negative Infinity

-10 / 0
Result: 
-Infinity

Step 5: Checking Numbers

isNaN()

isNaN("hello") // true
isNaN(5)       // false
---

Number.isFinite()

Number.isFinite(10)       // true
Number.isFinite(Infinity) // false

Step 6: Basic Math Operators (Revisited Deeply)

let a = 10;
let b = 3;

a / b   // 3.333...
a % b   // 1

Deep Understanding of % (Modulo)

Modulo is NOT just “remainder”.

It answers:

“What is LEFT after full division?”
Example: 
10 % 3

3 goes into 10 → 3 times

Remaining: 

1
---

Real Use Cases

  • Even / odd check → n % 2
  • Cycling values (games, UI)
  • Pagination systems
---

Step 7: Precision Problems (CRITICAL)

Floating point math can break expectations. 

0.1 + 0.2 !== 0.3

Solution approach: 

let result = (0.1 + 0.2).toFixed(2);
---

Step 8: Mental Model Summary

  • Numbers are stored in memory boxes
  • All numbers are floating point internally
  • Math operations read → compute → store
  • Some numbers are imprecise
  • Special values exist (NaN, Infinity)
