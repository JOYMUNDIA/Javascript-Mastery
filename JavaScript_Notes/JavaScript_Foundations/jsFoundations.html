From Zero Knowledge → Confident JavaScript Developer

Step 1: How JavaScript Runs in the Browser

JavaScript in the browser runs inside the browser’s engine. Examples:

  • Chrome → V8 engine
  • Firefox → SpiderMonkey
  • Safari → JavaScriptCore (Nitro)

When you open a webpage, the browser:

  1. Parses the HTML and CSS.
  2. Finds JavaScript code in <script> tags or linked files.
  3. Executes the JS line by line in the engine.

Think of it like this:

“HTML builds the page, CSS styles it, JS adds behavior and interactivity.”
Mini Challenge: Open any website, right-click → Inspect → Console. Type console.log("Hello from JS!").

Step 2: How JavaScript Runs in Node.js

Node.js allows JavaScript to run outside the browser on your computer.

  • It uses the V8 engine (same as Chrome) to execute JS.
  • Provides extra modules like fs for files, http for servers.
  • Perfect for building backend apps, scripts, and command-line tools.
// Save as hello.js
console.log("Hello from Node.js!");
// Run in terminal: node hello.js

Mental model: Node.js = JavaScript + your computer’s system APIs. Browser JS = JavaScript + browser APIs.

Mini Challenge: Install Node.js, write a small JS file that prints today's date and run it in terminal.

Step 3: Linking JavaScript to HTML

There are 3 main ways to include JS in your webpage:

1️⃣ Inline Scripts

<button onclick="alert('Clicked!')">Click Me</button>

Not recommended for big projects, but useful for tiny interactions.

2️⃣ Internal Scripts

<!DOCTYPE html>
<html>
<body>

<h1>Hello</h1>

<script>
  console.log("Internal JS runs here!");
</script>

</body>
</html>

3️⃣ External Scripts

<!DOCTYPE html>
<html>
<body>

<h1>Hello</h1>

<script src="script.js"></script>

</body>
</html>

External scripts keep your HTML clean and easier to maintain.

Mini Challenge: Create index.html and script.js, link them, and print “JS is linked!” in the console.

Step 4: Using DevTools Console

The browser DevTools console is your personal JS playground.

Open DevTools:

  • Chrome / Edge: Ctrl+Shift+I or F12
  • Firefox: Ctrl+Shift+K
  • Safari: Cmd+Option+C (enable Developer menu first)

Console Uses:

  • Print values: console.log("Hello")
  • Debug variables: console.log(variable)
  • Inspect objects: console.table(array)
  • Check types: console.log(typeof x)
  • Errors & warnings: console.error(), console.warn()

Example:

let age = 25;
console.log("User age:", age);
console.warn("This is just a warning!");
console.error("Something went wrong!");
Mini Challenge: Open DevTools, create 3 variables of different types, print them using console.log, and use console.table for an array.

Step 5: Mental Model – JS Environments

Think of JavaScript as a language that can “plug into” different environments:

  • Browser: manipulates webpage, DOM, CSS, and handles events
  • Node.js: manipulates files, servers, databases, command-line scripts
  • DevTools Console: testing ground, experiment, debugging tool

Key takeaway:

“Where JS runs determines what APIs it can use, but the core language works the same everywhere.”
Mastery Challenge: Open DevTools console, declare a variable, manipulate it, and compare it to running the same code in Node.js. Observe differences.
